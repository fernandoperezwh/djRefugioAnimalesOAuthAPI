Ventajas y desventajas de OAuth 2.0

Ventajas

  • Los tokens de acceso se vinculan a scopes particulares, lo que restringe los tipos de operaciones y datos a los que puede acceder la aplicación.
  • Los tokens de acceso caducan o se pueden revocar por lo que los efectos negativos podrían tener un impacto limitado.
  • Se tienen dos llaves client_id y client_secret que se obtiene al registrar una nueva aplicación desde el dashboard de desarrolladores. Estas se usan en lugar de usuario y contraseña para no comprometerlas.
  • Podemos mejorar la experiencia del usuario con el mismo OAuth usando el authorization code. El usuario final es redirigido y solo debe dar clic a un botón para autorizar.
  • Da soporte a diferentes escenarios de negocio dependiendo del tipo de consumidor que accede a los datos que expone la API.

Desventajas

  • Podria decirse que es extendido el paso para obtener un token de accesso. Sin embargo esto lo hace mas seguro.
    El flujo mas simple (Client credentials) requiere un paso extra, el cual es hacer el encode en base64 de `client_id:client_secret`. Con Api Key, tal cual te la key es la que usaras.
